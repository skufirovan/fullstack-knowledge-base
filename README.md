# KB Platform

Внутренняя платформа для базы знаний компании.

Проект представляет собой fullstack веб-приложение для хранения, просмотра и управления корпоративной документацией и статьями.
Платформа поддерживает систему ролей, разделы, markdown-статьи, загрузку изображений и контейнеризацию через Docker.

---

## Основные возможности

* Авторизация и аутентификация пользователей
* Система ролей (`admin`, `editor`, `user`)
* Создание и просмотр разделов
* Создание, редактирование и публикация статей
* Поддержка Markdown
* Загрузка изображений через S3-хранилище
* Медиатека с просмотром загруженных изображений
* Sidebar-навигация по категориям и статьям
* Docker-контейнеризация
* Reverse proxy через Nginx

## Запуск проекта

### 1. Клонирование репозитория

```bash
git clone https://github.com/skufirovan/fullstack-knowledge-base.git
cd fullstack-knowledge-base
```

---

### 2. Настройка backend

В директории `backend` необходимо создать `.env` файл на основе `.env.example`.

---

### 3. Запуск через Docker

```bash
docker compose up --build
```

После запуска приложение будет доступно по адресу:

```txt
http://localhost
```

## Особенности проекта

### Markdown

Статьи поддерживают markdown-разметку:

* заголовки;
* списки;
* таблицы;
* чек-листы;
* ссылки;
* форматирование текста.

---

### Загрузка изображений

Изображения загружаются напрямую в S3-хранилище через presigned URL.
После загрузки файлы подтверждаются сервером и становятся доступны в медиатеке.

---

### Docker-инфраструктура

Проект полностью контейнеризирован:

* PostgreSQL запускается в отдельном контейнере;
* Backend и Frontend собираются через Dockerfile;
* Nginx используется как reverse proxy;
* Prisma migrations и seed выполняются автоматически при запуске backend-контейнера.
